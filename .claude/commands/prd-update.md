你现在是产品经理角色。对照设计稿，找出已有 PRD 的缺失章节并补写。

## 输入规范

必须同时提供两个参数：
- `@设计稿` — HTML / JSX 原型文件（真理之源）
- `@docs/prds/xxx.md` — 已有 PRD 文件（要补全的目标）

## 执行流程

### 第一步：扫描设计稿，生成元素清单

读取设计稿文件，枚举所有顶层 UI 区域：
- HTML 文件：提取所有顶层结构 class（如 `.topbar` / `.activity-bar` / `.shell-panel`）
- JSX 文件：提取所有组件函数名

生成元素清单表格：

| # | 元素名 | 设计稿位置 | 描述 |
|---|--------|-----------|------|
| 1 | Activity Bar | `.activity-bar` | 左侧 48px 固定导航栏 |
| ... | | | |

### 第二步：读取已有 PRD，做覆盖率对比

读取已有 PRD 文件，逐项检查设计稿元素是否有对应章节：

```
输出格式：
  ✅ TopBar — 已覆盖 (#顶部操作栏)
  ✅ Pipeline Strip — 已覆盖 (#流水线步骤条)
  ❌ Activity Bar — PRD 中无对应章节
  ❌ Shell Panel — PRD 中无对应章节
  ❌ Command Palette — PRD 中无对应章节

缺失 3 项，已覆盖 11 项，覆盖率 79%
```

### 第三步：逐项询问用户决策

对每个缺失项，展示给用户并等待确认：
- ✅ **v1 补入** — 写入本期 PRD，生成对应功能章节和业务规则
- 🔜 **v2 延后** — 写入 PRD「范围外 / 后续版本」章节，不生成任务
- ❌ **不做** — 排除，不写入 PRD

### 第四步：补写缺失章节

对所有「v1 补入」的元素，按照 `docs/prds/_template.md` 格式补写功能章节，包括：
- `## 功能点 N: 元素名称` 二级标题
- 用户故事
- 业务规则（从设计稿推断，不确定的标 `[待确认]`）
- 异常场景

补写完后，将修改追加到已有 PRD 文件末尾（或插入合适位置），不覆盖原有内容。

### 第五步：输出覆盖率报告

```
✅ PRD 补全完成：docs/prds/xxx.md

新增章节：
  - #Activity Bar（功能点 12）
  - #Shell Panel（功能点 13）
  - #Command Palette（v2，写入范围外章节）

覆盖率：14/14（100%）

建议下一步：
  1. 人工审阅新增章节，把 [待确认] 项填实
  2. 跑 /prd-check 验证 PRD 完备性
  3. 跑 /plan 重新拆解任务
```

## 注意事项

- 不删除、不修改已有 PRD 内容，只追加缺失章节
- 业务规则从设计稿 CSS / 结构推断，不确定的一律标 `[待确认]`
- 补写后必须跑 `/prd-check` 验证
